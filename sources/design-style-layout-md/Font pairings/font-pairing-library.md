## Google Font Usage Guidelines (Universal AI Rules)

### 1) Default strategy: variable fonts first
- Prefer variable fonts as the primary choice for modern websites.
- Use one variable family with a wide weight range for UI consistency and performance.
- Keep fallback stacks clean (system sans/serif/mono) in case the chosen family is unavailable.

### 2) Content-safe typography rules
- For paragraph/content text, prioritize stable families with high readability:
  - business
  - calm
  - monospace (for code/data tone)
  - serif
  - sans serif
- Decorative or novelty styles should not be used for long paragraph blocks.
- Keep body text in a predictable style even if headlines are expressive.

### 3) Creative/decorative font flexibility
- Allow decorative fonts for brand voice, campaign identity, and hero headlines.
- Constrain decorative usage to high-impact zones: logo lockups, hero titles, section openers, event badges.
- Pair decorative display with a calm readable body font (usually sans serif or serif).

### 4) Keyword-triggered style mapping (common website patterns)
- Use user keywords as style triggers, then apply decorative fonts only where appropriate.

| User keyword/theme | Recommended style direction | Typical placement |
|---|---|---|
| business, corporate, finance, consulting | neutral sans/serif, restrained contrast | full site typography |
| calm, wellness, minimal, meditation | soft sans or humanist serif | full site typography |
| mono, developer, terminal, technical | monospace + neutral sans | code blocks, labels, hero accent |
| luxury, fashion, editorial, sophisticated | high-contrast serif or elegant display + clean sans | hero/display + section headers |
| playful, childlike, happy | rounded/friendly display + simple sans | campaign hero, cards, banners |
| artistic, handmade, crafty, calligraphy | script/calligraphic display + readable sans | logo, hero, invitations, feature titles |
| vintage, retro, art deco, blackletter | period-style display + neutral body font | hero, badges, promo headings |
| futuristic, techno, pixel, stencil | geometric/tech display + neutral body font | launch pages, gaming/tech promos |
| halloween, spooky, horror | distressed or rough display + readable body | seasonal hero, event modules |
| christmas, valentines, lunar new year, diwali, hanukkah, kwanzaa | festive cultural display + stable body font | event headers, promo banners, cards |
| indian, diwali, south-asian festive | decorative Indic-friendly display + neutral text font | event campaign headers, celebration modules |

### 5) Safety guardrails for AI generation
- Never apply highly decorative fonts to dense body copy, long-form articles, or critical UI flows.
- Keep at most 1 primary display font + 1 content font pair per page section.
- If readability drops, downgrade decorative fonts to title-only and switch body to business/calm serif/sans.
- Prioritize accessibility and comprehension over stylistic novelty.

### 6) Do / Don't Quick Rules

#### Do
- Use variable fonts first for flexibility and performance.
- Keep to 1 primary display font + 1 content font per section.
- Use sans/serif/business/calm/mono styles for paragraphs and long reads.
- Reserve decorative fonts for hero headlines, badges, event banners, and brand accents.
- Match decorative styles to user keywords (e.g., diwali, halloween, handmade, techno) in limited zones.
- Validate readability on desktop, mobile, and multilingual content.

#### Don't
- Don't use decorative/script fonts for dense body text, forms, tables, or critical instructions.
- Don't mix too many expressive fonts in one viewport.
- Don't apply all-caps decorative styles to long multi-line headings.
- Don't sacrifice legibility and contrast for novelty.
- Don't force one decorative font across all sections.
- Don't skip fallback stacks (`sans-serif`, `serif`, `monospace`).
