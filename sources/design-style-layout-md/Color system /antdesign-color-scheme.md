# Ant Design Color Scheme (Frontend + App)

This document is based on Ant Design color specification and uses official `@ant-design/colors` palette values with matching names, tokens, usage categories, and dark mode mappings.

## Usage Categories (Ant Design Spec)
- **System-level color system**: base color palettes, neutral palette, and data-visualization-related palette foundations.
- **Product-level color system**: brand color, functional colors, and neutral UI tokens for text/background/border.

## 5) Frontend Usage Guidance
- Prefer semantic/product tokens (`token.colorPrimary`, `token.colorText`, etc.) in components over hardcoded hex values.
- Use base palettes (`red-*`, `blue-*`, etc.) for illustration, charts, and extended thematic needs.
- Keep functional color semantics consistent: success=green, warning=gold/yellow, error=red, info/link=blue.
- Validate contrast in both light and dark themes for all text-background and icon-background combinations.
