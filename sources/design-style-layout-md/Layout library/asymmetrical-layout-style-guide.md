# Asymmetrical Layout Style Guide

## Overview
This guide defines an asymmetrical layout system for AI vibe coding website design tools.  
It is built for creative, image/video-heavy websites that need a modern, dynamic, and experience-focused presentation.

Asymmetry here means intentional imbalance, not visual chaos.  
The goal is to create energy, focal contrast, and clear narrative flow using unequal element sizes and positions.

---

## Why Asymmetry
Use asymmetrical composition to:
- Create visual interest and movement
- Direct attention to key media or CTA areas
- Make layouts feel contemporary, bold, and distinctive
- Support storytelling through pacing and contrast

Asymmetry should still maintain **visual balance** through spacing, hierarchy, and weight distribution.

---

## General Layout Direction
- **Style**: Modern, minimal, creative, scattered freeform, clear
- **Tone**: Bold, dynamic, playful-but-professional
- **Color**: No fixed palette; allow monochrome/grayscale base with optional bright accent blocks
- **Fonts**: Do not prescribe font family; prioritize readability and modern scale contrast
- **Icons**: No icon dependency
- **Images/Videos**: High-quality, striking, and relevant
- **Text**: Concise and impactful, with proper hierarchy
- **Layout Engine**: Fluid responsive (no rigid fixed-canvas behavior)

---

## Visual Balance: Symmetry vs Asymmetry

### Symmetry (Reference)
Symmetry provides order, familiarity, and stability through mirrored or repeated structure.

### Asymmetry (Primary)
Asymmetry creates modern energy through contrast:
- different block sizes
- unequal placement
- varied media proportions
- layered depth and staggered anchors

To keep asymmetry usable, balance visual weight using:
- size and scale
- contrast and density
- negative space
- alignment anchors

---

## Core Asymmetrical System Rules

### 1) Unequal but Intentional Composition
- Use mixed block scales in each section (large + medium + small)
- Avoid mirrored left/right layouts
- Keep one dominant focal element per section

### 2) Visual Weight Balancing
Balance heavier elements (large media, dense text) with:
- lighter modules
- open space
- offset placement

### 3) Layering and Overlap
- Layer elements when it improves focus and storytelling
- Overlap can be used for depth, but primary content must remain legible
- Keep overlap limited and intentional, not decorative noise

### 4) Negative Space as Structure
- Use empty space to separate focal moments
- Do not fill every area
- Let breathing room create rhythm between heavy sections

### 5) Section Anchoring
- Even in freeform layouts, keep anchor alignments
- Maintain consistent small outer left/right spacing
- Keep section rhythm coherent across breakpoints

---

## Accent Block Guidance (Spotty Bright Shapes)
Allowed:
- Large bright stripe-like background accent
- Large circular accent block
- Spotty accent placements to create contrast rhythm

Rules:
- Use accents as background support, not content containers
- Do not require fixed brand color values
- Keep accents sparse and strategic
- Ensure media and text remain primary focal layers

---

## Media-Heavy Storytelling Use Cases
Use this layout when users request:
- many images/videos in one site
- creative storytelling pages
- fun, modern presentation websites
- campaign, portfolio, lifestyle, fashion, culture, and event experiences

Recommended structure:
1. Hero focal composition (large media + concise headline)
2. Asymmetrical media narrative sections
3. Layered transition sections with accent blocks
4. CTA-driven closure section

---

## Responsive / Fluid Behavior
- Use fluid width behavior across screen sizes
- Rebalance asymmetry per breakpoint (not just stack everything)
- Preserve hierarchy on tablet/mobile:
  - primary focal media stays dominant
  - secondary elements reflow without losing narrative order
- Keep spacing in `rem` units for scalable rhythm

---

## Content and Hierarchy Rules
- Large headline for section intent
- Short supporting text (1-3 lines preferred)
- Clear CTA labels
- Avoid long text walls in high-impact visual sections
- Keep copy density lower in image-dominant blocks

---

## AI Output Requirements
When generating this style with AI, enforce:
- true asymmetrical composition (non-mirrored)
- strong visual balance despite uneven structure
- image/video-forward hierarchy
- optional layered overlaps with readability preserved
- sparse bright accent background blocks (stripe/dot style)
- concise text hierarchy
- fluid responsive behavior with `rem` spacing

Reject outputs that:
- look like rigid symmetrical grids
- feel random with no focal hierarchy
- over-layer elements until readability drops
- overuse accent shapes so content loses priority
- collapse into generic stacked cards without asymmetrical logic
